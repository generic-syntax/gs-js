Generic-Syntax for Object Mapping

Lack of JSON and XML

GS
<http://schema.org/Article identifier='5Pl8d4' {
  author= "Author name"
  datePublished= "2020-1-1"
  image= <http://schema.org/imageObject {
  	url= "http://example.com/image.png"
  	height=600
  	width=800
  }>
  publisher= <http://schema.org/Organization name='Publisher name' {
  	logo=<http://schema.org/imageObject url='http://example.com/logo.png'>
  }>
}>
JSON-LD
{
	"@type": "http://schema.org/Article",
	"identifer": "5Pl8d4"
	"author": "Author name",
	"datePublished": "2020-1-1",
	"image": {
		"@type": "http://schema.org/imageObject",
		"url": "http://example.com/image.png",
		"height": 600,
		"width": 800
	},
	"publisher": {
		"@type": "http://schema.org/Organization",
		"name": "Publisher name",
		"logo": {
			"@type": "http://schema.org/imageObject",
			"url": "http://example.com/logo.png"
		}
	}
}

JSON-LD is today one of the best for serializing a typed objects graph. But the incapability to name objects in JSON has two flaws:

  • The @type property affects readability.
  • The absence of named header enriched with attributes prevents efficient event driven parsing for directly instantiate typed objects (and for avoid to instantiate costly intermediate generic objects).

On the over side, XML lacks of a key-node pair structure and is not well suited for this purpose.

GS solves these problems by combining JSON object properties and XML named object features.

GS-OM language

GS-OM is a secured library for building and serializing a complex and recursive graph of typed objects. At parsing stage, according to its configuration, typed objects are instanciated, properties are assigned and methods explicitly authorized can be called.

<?OM>
<User id='2197' {
  <%id"JOM">
  name= "John Merting"
  chief= <%ref"MAD">
  groups= [
    <%ref"mgrs">
  ]
}>
<User id='2123' {
  <%id"MAD">
  name= "Marc Dublof"
  groups=[
    <Group id='36564' {
      <%id"admins">
      name= "Administrators"
    }>
  ]
  addTemporaryDelegation = <%call [
	<%ref"JOM">
  	<Date day='2019-12-22'>
  	14
  ]>
}>
<Group id='35' {
  <%id"mgrs">
  name= "Managers"
}>
<%with ref=admins {
  addAlias= <%call ["AdminSys"]>
}>

GS-OM use instruction GS node type <%> for link objects and call methods at the building step.

GS-OM is a project, currently not implemented.
